Features

Everything You Need,
Nothing You Don't

Full S3 compatibility with a built-in dashboard, security hardening, and features that competitors charge enterprise pricing for.

Core S3 & Dashboard 09

Built-in Dashboard

A full React UI embedded in the single binary — no separate web server, no extra container. Run the whole platform from the browser.

File browser IAM & policies Audit trail Search Versioning & diff Dark mode

5 Rounds of Security

60+ hardening fixes: AES-256-GCM, SigV4 auth, path-traversal & SSRF protection, rate limiting, CSP headers.

S3 Select (SQL)

Query CSV/JSON objects with SQL without downloading — SELECT, WHERE, LIKE, AND/OR, LIMIT, right from the S3 API.

Versioning & WORM

Per-bucket versioning with a git-like diff viewer, legal hold & retention (GOVERNANCE/COMPLIANCE), one-click rollback.

FUSE Mount

Mount any bucket as a local directory — LRU block cache, metadata caching, SigV4 auth, read/write support built in.

IAM & OIDC/SSO

Users, groups & S3-compatible policy evaluation, with OIDC login (Google, Keycloak, Auth0) and role mapping.

Full-text Search

Search every bucket by key, content type, and tags — in-memory index with LRU eviction and incremental updates.

Event Notifications

Webhooks, Kafka, NATS, Redis, AMQP, PostgreSQL & Elasticsearch — with per-bucket prefix/suffix event filtering.

Lambda Triggers

Invoke external functions on S3 events — webhook-based with filtering, optional response storage, worker-pool dispatch.

Distributed & Durability 06

Erasure Coding

Reed–Solomon encoding across multiple disks with a background healer that auto-reconstructs degraded objects. Lose disks without losing data — no RAID controller required.

Reed–Solomon Configurable shards Background healer Survives disk loss

Raft Clustering

Strongly-consistent metadata via Raft consensus, with consistent-hash placement, quorum reads/writes, and failover.

Active-Active Replication

Multi-site sync with vector-clock conflict resolution, plus one-way async push. Keep regions in step.

Scales to 100M Objects Benchmarked

List a prefix with millions of keys without slowdown — served from a sorted metadata index, never a full-bucket scan.

~0.7 msper page — flat from 1K to 100M objects

Tiering & Backup

Transparent hot/cold tiering (local or remote S3) with promote-on-access, plus a full/incremental backup scheduler.

Small-file Packing Beta

Pack millions of tiny objects into large append-only volumes (independent zstd frames + a byte-offset index) instead of one file each — no per-file inode/syscall overhead at scale, with background compaction to reclaim space.

Intelligence & Ops 05 · all new

Semantic / Vector Search New

A built-in RAG store — no external vector database. Objects are embedded via any OpenAI-compatible model and served as cosine-similarity search, all inside the single binary.

Ollama llama.cpp OpenAI Cosine kNN No external DB

Migrate from MinIO New

Point the dashboard wizard at any S3 source, pick buckets, import with live progress — streamed, auto-retried, cancellable, no AWS SDK.

Auto-Update New

Opt-in daily release check with a dashboard banner, plus checksum-verified self-update. Your data is never touched.

Cost Estimator New

See what your data would cost on S3, GCS, R2, B2, or Wasabi — storage plus egress. Self-hosting is egress-free.

Bucket Snapshots New

Git-for-buckets: commit a bucket's state, diff against live, roll back in one click — even resurrects deleted objects. lakeFS in a single binary.

Get Started

Up and Running in 30 Seconds

One command. That's it. No databases, no dependencies, no configuration files.

# Pull and run with Docker
docker run -d \
  --name vaults3 \
  -p 9000:9000 \
  -e VAULTS3_ACCESS_KEY=myadmin \
  -e VAULTS3_SECRET_KEY=mysupersecret \
  -v vaults3-data:/data \
  -v vaults3-meta:/metadata \
  eniz1806/vaults3

# S3 API → http://localhost:9000
# Dashboard → http://localhost:9000/dashboard/
# Metrics → http://localhost:9000/metrics
